# Design: v2.2 — V2 cutover (ship the unshipped, toss the legacy)

## Overview

The v2.0 redesign was specified across two parallel architectures: a new V2 schema/policy module (`schemas/config-v2.ts`, `policy-v2/`) describing a flat, simplified security model, and a legacy schema/policy module (`schemas/config.ts`, `policy.ts`) that the production code actually wired through. The V2 design never reached `loadConfig()` — it sits in the codebase as scaffolding the docs claim is canonical but the code ignores.

This design **ships the V2 design as canonical** and **deletes the legacy entirely**. No fallback, no compat shim, no "legacy was X" framing in docs. v2.2.0 ships breaking config changes; users run `skilltap migrate` once and the app loads against the new schema.

This is a **forward sweep**. Every doc rewrite describes the V2 surface as if it always was. Every code change deletes a legacy module rather than aligning to it.

### What ships in v2.2.0

**Canonical security config** — flat 3-key `[security]` block:
```toml
[security]
scan = "static"           # semantic | static | none
on_warn = "install"       # prompt | fail | install
trust = ["github.com/my-corp/*", "internal-tap"]   # glob patterns
```

**Operational scanner config** — new `[scanner]` block (separates "what runs the scan" from "what the policy is"):
```toml
[scanner]
agent_cli = ""            # path/name of semantic-scan CLI (claude, codex, ollama, …)
ollama_model = ""         # model name when agent_cli = "ollama"
threshold = 5             # 0-10, semantic scan severity threshold
max_size = 51200          # bytes, skill size limit
```

**Wired policy resolver** — `composePolicy` lives in `packages/core/src/policy/` (promoted from `policy-v2/`), with `--agent`/`SKILLTAP_AGENT`/`[agent]` plumbing stripped (the v2.0 redesign already retired agent mode).

**Removed for good** — no fallback, no alias:
- `[security.human]` / `[security.agent]` per-mode blocks
- `[[security.overrides]]` array, `preset =` field, `TrustOverride` type, `SECURITY_PRESETS`
- `require_scan` policy key (use `on_warn = "fail"` for hard-fail behavior)
- `[agent-mode]` block, `[agent]` block, `--agent` flag, `SKILLTAP_AGENT` env var (redesign legacy — also fully gone now from `policy-v2/compose.ts`)
- `[registry] allow_npm` deprecated key
- `verify` CLI command (use `doctor skill <path>`)
- `linkSkill` core function (CLI surface was already gone)
- `httpAdapter` source adapter (HTTP taps long gone — adapter is unreachable)
- `searchSkillsRegistry` deprecated wrapper, `marketplaceSourceToRepo` orphan
- `installed.json` / `plugins.json` writes from `info.ts` legacy fallback
- All `link`/`unlink`/`verify`/`enable`/`disable` references in completions, hints, fixtures

**Migration path:** `skilltap migrate` translates v0.x AND legacy-v2.x configs → V2 shape. After migrate, `loadConfig` reads V2. Pre-migrate users hit a hard error pointing at `skilltap migrate`. No automatic fallback, no "tried both shapes" parser branching.

### Unit 1.1: Define the canonical V2 schema in `schemas/config.ts`

**File**: `packages/core/src/schemas/config.ts` (rewritten — replaces both legacy `ConfigSchema` and the unshipped `ConfigV2Schema`)

```typescript
import { z } from "zod/v4";

// ────────────────────────────────────────────────────────────────────────
// Enum constants — single source of truth for valid values.
// ────────────────────────────────────────────────────────────────────────

export const SCAN_MODES = ["semantic", "static", "none"] as const;
export const ON_WARN_MODES = ["prompt", "fail", "install"] as const;
export const SCOPE_VALUES = ["", "global", "project"] as const;
export const AUTO_UPDATE_MODES = ["off", "patch", "minor"] as const;
export const SHOW_DIFF_MODES = ["full", "stat", "none"] as const;
export const SOURCE_TYPES = ["tap", "git", "npm", "local"] as const;

// ────────────────────────────────────────────────────────────────────────
// [security] — policy. Three keys.
// ────────────────────────────────────────────────────────────────────────

export const SecurityConfigSchema = z
  .object({
    /** Default scan mode. CLI flag --deep upgrades to "semantic" per-call. */
    scan: z.enum(SCAN_MODES).default("static"),
    /** What happens when a scan produces warnings. */
    on_warn: z.enum(ON_WARN_MODES).default("install"),
    /**
     * Glob patterns matched against tap name OR full source URL.
     * A matching source skips the scan entirely (trusted = true).
     */
    trust: z.array(z.string()).default([]),
  })
  .prefault({});

// ────────────────────────────────────────────────────────────────────────
// [scanner] — operational config. What CLI to invoke, size limits, etc.
// ────────────────────────────────────────────────────────────────────────

export const ScannerConfigSchema = z
  .object({
    /** Path or name of the semantic-scan agent CLI (claude, codex, ollama, custom path). */
    agent_cli: z.string().default(""),
    /** Model name when agent_cli = "ollama". Ignored otherwise. */
    ollama_model: z.string().default(""),
    /** Risk threshold for semantic scan (0–10). Chunks scoring ≥ this are flagged. */
    threshold: z.number().int().min(0).max(10).default(5),
    /** Skill size limit in bytes. Skills larger than this are rejected. */
    max_size: z.number().int().default(51200),
  })
  .prefault({});

export const UpdatesConfigSchema = z
  .object({
    auto_update: z.enum(AUTO_UPDATE_MODES).default("off"),
    interval_hours: z.number().int().default(24),
    skill_check_interval_hours: z.number().int().default(24),
    show_diff: z.enum(SHOW_DIFF_MODES).default("full"),
  })
  .prefault({});

export const TelemetryConfigSchema = z
  .object({
    enabled: z.boolean().default(false),
    notice_shown: z.boolean().default(false),
    anonymous_id: z.string().default(""),
  })
  .prefault({});

export const RegistrySourceSchema = z.object({
  name: z.string(),
  url: z.string(),
});

export const RegistryConfigSchema = z
  .object({
    enabled: z.array(z.string()).default(["skills.sh"]),
    sources: z.array(RegistrySourceSchema).default([]),
  })
  .prefault({});

export const ConfigDefaultsSchema = z
  .object({
    also: z.array(z.string()).default([]),
    yes: z.boolean().default(false),
    scope: z.enum(SCOPE_VALUES).default(""),
  })
  .prefault({});

export const TapEntrySchema = z.object({
  name: z.string(),
  url: z.string(),
});

export const ConfigSchema = z.object({
  defaults: ConfigDefaultsSchema,
  security: SecurityConfigSchema,
  scanner: ScannerConfigSchema,
  registry: RegistryConfigSchema,
  taps: z.array(TapEntrySchema).default([]),
  updates: UpdatesConfigSchema,
  telemetry: TelemetryConfigSchema,
  builtin_tap: z.boolean().default(true),
  verbose: z.boolean().default(true),
  default_git_host: z.string().default("https://github.com"),
});

export type SecurityConfig = z.infer<typeof SecurityConfigSchema>;
export type ScannerConfig = z.infer<typeof ScannerConfigSchema>;
export type UpdatesConfig = z.infer<typeof UpdatesConfigSchema>;
export type TelemetryConfig = z.infer<typeof TelemetryConfigSchema>;
export type RegistryConfig = z.infer<typeof RegistryConfigSchema>;
export type RegistrySource = z.infer<typeof RegistrySourceSchema>;
export type ConfigDefaults = z.infer<typeof ConfigDefaultsSchema>;

**Implementation Notes**:
- This schema replaces both the legacy `ConfigSchema` (with `[[security.overrides]]` etc.) and the unshipped `ConfigV2Schema`. There is now ONE canonical config schema in the codebase.
- `[agent]` block (from the unshipped V2 schema) is dropped — `--agent` was retired in v2.0; no resurrection.
- `[registry].allow_npm` deprecated key is dropped (no producer or consumer).
- `[[taps]].type`, `[[taps]].auth_token`, `[[taps]].auth_env` from the legacy schema are dropped — all taps are git-only now (HTTP removed); auth flows through git CLI directly.
- `defaults.yes` retained (it was on legacy `defaults`, useful for "always pass --yes" preference).
- The new `[scanner]` block separates operational config from policy. The V2 spec only had 3 security keys, but the scanner needs to know `agent_cli`/`ollama_model`/`threshold`/`max_size` to function. Putting them in a sibling block keeps `[security]` clean per V2 intent without breaking semantic scanning.

**Acceptance Criteria**:
- [ ] `ConfigSchema.parse({})` produces a config with `security.scan = "static"`, `security.on_warn = "install"`, `security.trust = []`, `scanner.threshold = 5`, etc.
- [ ] `ConfigSchema.parse({ security: { trust: ["github.com/me/*"] } })` succeeds.
- [ ] `ConfigSchema.parse({ security: { overrides: [...] } })` produces a parsed config where `overrides` is silently absent (Zod strips unknown keys by default — but Unit 1.6 adds explicit detection + hard-fail for legacy shapes before this point).
- [ ] No exported symbol `TrustOverrideSchema`, `PRESET_VALUES`, `SECURITY_PRESETS`, `AgentConfigSchema`, `ConfigV2Schema`, `SecurityConfigV2Schema` remains.
- [ ] `bun test packages/core/src/schemas/config.test.ts` passes after test rewrites.

import type { Config } from "../schemas/config";
import { ok, type Result, UserError } from "../types";
import { isTrusted } from "./trust-glob";
import type { CliFlags, EffectivePolicy, SourceForPolicy } from "./types";

function resolveYes(flags: CliFlags): boolean {
  if (flags.noYes === true) return false;
  return flags.yes === true;
}

function resolveScanMode(config: Config, flags: CliFlags): EffectivePolicy["scanMode"] {
  if (flags.deep === true) return "semantic";
  return config.security.scan;
}

function resolveOnWarn(config: Config, flags: CliFlags): EffectivePolicy["onWarn"] {
  if (flags.noStrict === true) return config.security.on_warn;
  if (flags.strict === true) return "fail";
  return config.security.on_warn;
}

function resolveScope(config: Config, flags: CliFlags): EffectivePolicy["scope"] {
  if (flags.project === true) return "project";
  if (flags.global === true) return "global";
  return config.defaults.scope;
}

export function composePolicy(
  config: Config,
  flags: CliFlags,
): Result<EffectivePolicy, UserError> {
  return ok({
    yes: resolveYes(flags),
    scope: resolveScope(config, flags),
    also: config.defaults.also,
    scanMode: resolveScanMode(config, flags),
    onWarn: resolveOnWarn(config, flags),
    skipScan: flags.skipScan === true,
    trusted: false,
  });
}

export function composePolicyForSource(
  config: Config,
  flags: CliFlags,
  source: SourceForPolicy,
): Result<EffectivePolicy, UserError> {
  const base = composePolicy(config, flags);
  if (!base.ok) return base;
  if (isTrusted(config.security.trust, source)) {
    return ok({ ...base.value, trusted: true, scanMode: "none" });
  }
  return base;
}
```

### Unit 1.4: Replace `loadConfig()` to parse V2; hard-fail on legacy shapes

**File**: `packages/core/src/config.ts:89-129`

```typescript
export async function loadConfig(): Promise<Result<Config, UserError>> {
  await ensureDirs();
  const path = configPath();
  const exists = await Bun.file(path).exists();
  if (!exists) return ok(ConfigSchema.parse({}));

  const text = await Bun.file(path).text();
  let raw: unknown;
  try {
    raw = parseToml(text);
  } catch (e) {
    return err(new UserError(`Failed to parse config.toml: ${(e as Error).message}`));
  }

  // Hard-fail on legacy shapes — no silent fallback.
  const legacyDetection = detectLegacyConfig(raw);
  if (legacyDetection !== null) {
    return err(
      new UserError(
        `Legacy config detected (${legacyDetection}). Run \`skilltap migrate\` to upgrade to the v2.2 config schema.`,
        { hint: "skilltap migrate" },
      ),
    );
  }

  return parseWithResult(ConfigSchema, raw as Record<string, unknown>, "config.toml");
}

// Returns the name of the first legacy marker found, or null if none.
function detectLegacyConfig(raw: unknown): string | null {
  if (!raw || typeof raw !== "object") return null;
  const r = raw as Record<string, unknown>;
  const sec = r.security as Record<string, unknown> | undefined;

  if (sec && typeof sec === "object") {
    if ("human" in sec) return "[security.human]";
    if ("agent" in sec) return "[security.agent]";
    if ("overrides" in sec) return "[[security.overrides]]";
    if ("require_scan" in sec) return "security.require_scan";
    if ("agent_cli" in sec) return "security.agent_cli";
    if ("ollama_model" in sec) return "security.ollama_model";
    if ("threshold" in sec) return "security.threshold";
    if ("max_size" in sec) return "security.max_size";
  }
  if ("agent-mode" in r) return "[agent-mode]";
  if ("agent" in r) return "[agent]";

  return null;
}
```

**Implementation Notes**:
- The `detectLegacyConfig()` function is the gate. It looks for any v0.x or pre-v2.2 marker and returns the offending key. The error message tells the user exactly which marker tripped it.
- After `migrate` runs, the user's config has `[scanner].agent_cli` etc. (not `[security].agent_cli`), so the detector's `[security].agent_cli` check fires only on un-migrated configs.
- Default-config-template (Unit 1.10) writes the V2 shape so a fresh install parses cleanly with no migration.

**Acceptance Criteria**:
- [ ] `loadConfig()` returns `ok(...)` for a clean V2 config.
- [ ] `loadConfig()` returns `err(...)` with hint `"skilltap migrate"` for a config containing `[security.human]`, `[[security.overrides]]`, `[agent-mode]`, or `security.require_scan`.
- [ ] Empty config dir → `ok(ConfigSchema.parse({}))` with all defaults.

- **`[scanner]` block** (operational, separate from policy): `agent_cli`, `ollama_model`, `threshold`, `max_size`.

- **Smart scope default**: inside a git repo, `install` defaults to `project`; outside, `global`.
- **`Output` interface**: all output goes through `setupOutput(args)` in CLI commands.
- **HTTP registry adapter removed** — taps are git-only.

## Risks

### R1: Users with custom `[[security.overrides]]` lose their preset choice on migrate
Migrate translates `preset = "none"` (trust) cleanly. `preset = "relaxed"`/`"standard"`/`"strict"` is dropped with a warning because the V2 model has no equivalent. **Mitigation**: warning text gives the user the explicit `scan`/`on_warn` values they would have gotten under the preset, so they can re-set them by hand. Document in changelog.

### R2: `require_scan = true` users lose hard-skip-prevention
The legacy `require_scan` blocked `--skip-scan`. V2 has no equivalent. **Mitigation**: migrate emits a warning suggesting `on_warn = "fail"`. If users genuinely need skip-prevention, that's a future feature (small follow-up: re-introduce `--skip-scan` as a no-op when `on_warn = "fail"`).

### R3: Round-trip migrate test depends on `loadConfig` reading the migrated file from the same `XDG_CONFIG_HOME` the test wrote it to
Test must use `createTestEnv()` to isolate the env, and pass through `XDG_CONFIG_HOME` overrides correctly. `loadConfig()` reads from the env-derived path, so the test must set the env before calling. **Mitigation**: existing `createTestEnv()` does this; just use it.

### R4: The agent-mode plumbing in `policy-v2/compose.ts` was the V2 design's intent — removing it deviates from the original V2 spec
The V2 schema's `[agent]` block was scaffolded BEFORE the v2.0 redesign decided to retire `--agent` entirely. The contradictions: `[agent]` block in V2 schema vs `--agent` removed in v2.0 narrative. Choosing one is the call. **Decision**: drop `[agent]` block; the v2.0 redesign's "no agent mode" win is the more recent design intent. Updates to V2 schema (Unit 1.1) reflect this.

### R5: Foundation doc rewrites (Unit 4.2 SPEC especially) are large
SPEC.md is 158k. Cutting ~100k of legacy content means a careful edit. **Mitigation**: spawn a dedicated Sonnet agent for Unit 4.2 with the explicit mandate to delete the legacy block and promote the canonical section. Cross-doc anchor sweep after.

### R6: Deleting `policy.ts` while CLI commands import from it
Strict ordering required: Unit 1.2 promotes `policy-v2/`, Unit 1.3 rewires callers, ONLY THEN Unit 1.11 deletes the legacy file. Build break if order slips. **Mitigation**: each Phase 1 unit has a `bun run build` check before committing.

### R7: Rename `loadInstalled`/`saveInstalled` → `loadSkillState`/`saveSkillState` touches many call sites
Mechanical but error-prone if missed. **Mitigation**: Unit 3.2 grep validation catches misses.
